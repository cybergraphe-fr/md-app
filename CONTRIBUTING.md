# Contributing

## Development setup

1. Install Go 1.25.x and Node.js 22+.
2. Install frontend dependencies from web/ with npm ci.
3. Start local development with make dev.

## Pull requests

1. Open an issue first for large changes.
2. Keep changes focused and documented.
3. Add or update tests when behavior changes.
4. Run make check before opening a pull request.
5. Fill in the pull request template completely.

## Commit quality

- Prefer small, reviewable commits.
- Do not mix unrelated refactors with functional changes.
- Preserve security hardening and deployment safety.

## Reporting security issues

Do not open public issues for vulnerabilities. Follow SECURITY.md instead.
